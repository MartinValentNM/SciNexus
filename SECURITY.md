# Security Policy

## Sensitive Data

Do not store the following in the repository:

- API keys  
- passwords  
- `.env` files  
- user data in the `users/` directory  
- local databases (`*.db`)  
- exports and backups  

## Reporting a Security Issue

Report security issues privately to the repository maintainer.  
Do not include secrets or reproduction data containing non‑public documents in public issues.

## Citlivá data

Do repozitáře neukládej:

- API klíče
- hesla
- `.env`
- uživatelská data ve složce `users/`
- lokální databáze `*.db`
- exporty a zálohy

## Nahlášení problému

Bezpečnostní problém nahlaš soukromě správci repozitáře. Do veřejného issue nevkládej tajné údaje ani reprodukční data obsahující neveřejné dokumenty.
